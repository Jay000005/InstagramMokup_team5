# index.html 왼쪽 사이드바 → 공통 사이드바 교체 가이드

## 현재 구조 정리

| 구분 | index.html (현재) | 공통 (sidebar.html / bottom-nav.html) |
|------|-------------------|----------------------------------------|
| **사이드바** | 인라인 `<aside class="sidebar">` (feed.css 스타일) | `sidebar.html` → `#sidebar`에 삽입 (sidebar.css) |
| **클래스** | `.sidebar`, `.logo`, `.sidebar-nav`, `.menu`, `.menu-item` | `.sidebar`, `.sidebar__logo`, `.sidebar__nav`, `.sidebar__item` |
| **모바일 하단** | 인라인 `<nav class="mobile-bottombar">` | `bottom-nav.html` → `#bottom-nav`에 삽입 (mobile.css) |
| **스크립트** | script.js (테마 + **sidebar/bottom-nav 로드** + 피드/스토리 로직) | 이미 `#sidebar`, `#bottom-nav` 넣으면 script.js가 fetch로 채움 |

index에는 `#sidebar`, `#bottom-nav` 플레이스홀더가 없고, explore/reels처럼 공통 HTML을 쓰지 않고 인라인으로만 되어 있음.

---

## 결론: “교체한 다음에 고치기”가 낫습니다

- **먼저 공통 사이드바로 교체**해서 `index`도 `sidebar.html` / `bottom-nav.html` / `sidebar.css` / `mobile.css` 기준으로 맞추고,
- **그 다음** index에만 있던 기능(예: 호버 시 64px→244px 넓어지는 동작, 메뉴 순서, 아이콘 등)을 **공통 파일(sidebar.html, sidebar.css)** 쪽에 반영하는 편이 좋습니다.

한 번만 공통을 보강하면 explore, reels, index 모두에 적용됩니다.

---

## 1. index.html 수정

### 1-1. head에 CSS 추가

공통 사이드바·하단 nav 스타일을 쓰려면 다음 두 개를 넣습니다.

```html
<link rel="stylesheet" href="styles/sidebar.css" />
<link rel="stylesheet" href="styles/mobile.css" />
```

- **sidebar.css**: 공통 왼쪽 사이드바 (`.sidebar`, `.sidebar__logo`, `.sidebar__nav`, `.sidebar__item` 등)
- **mobile.css**: `#bottom-nav`, `.bottom-nav` (모바일에서만 보이는 하단 탭)

`general.css`, `feed.css` 순서는 그대로 두고, 그 앞이나 뒤에 위 두 줄만 추가하면 됩니다.

### 1-2. 왼쪽 사이드바를 플레이스홀더로 교체

**삭제할 것:**  
`<aside class="sidebar">` 부터 `</aside>` 까지 전체 (로고 + nav 메뉴 전부).

**넣을 것:**

```html
<div id="sidebar"></div>
```

위치는 **모바일 상단바 다음, `.main-container` 바로 위**가 좋습니다.  
즉, `</header>` 다음에 `<div id="sidebar"></div>` 넣고, 그 다음에 `<div class="main-container">` 오면 됩니다.

### 1-3. 모바일 하단바를 플레이스홀더로 교체

**삭제할 것:**  
`<nav class="mobile-bottombar">...</nav>` 전체.

**넣을 것:**

```html
<div id="bottom-nav"></div>
```

`</body>` 직전에 두면 됩니다. (기존 `.main-container` 닫는 `</div>` 다음)

이렇게 하면 `script.js`의 기존 로직이 `#sidebar`에 `sidebar.html`, `#bottom-nav`에 `bottom-nav.html`을 넣어줍니다.

## 2. feed.css 수정 (레이아웃·중복 제거)

### 2-1. 공통 사이드바는 fixed이므로 본문에 여백 주기

`sidebar.css`의 `.sidebar`는 `position: fixed; width: 244px` 이라서, 본문이 겹치지 않으려면 `.main-container`에 왼쪽 여백을 줘야 합니다.

**추가할 것 (데스크톱 구간):**

```css
@media (min-width: 1024px) {
  .main-container {
    padding-left: 244px;  /* 또는 256px 등 디자인에 맞게 */
  }
}
```

이미 `@media (min-width: 1024px)` 블록이 있으면 그 안에 `padding-left`만 넣어도 됩니다.

### 2-2. 기존 “인라인 사이드바” 전용 스타일 정리

아래는 **index 인라인 사이드바용**이라서, 공통으로 넘어간 뒤에는 제거하거나 주석 처리해도 됩니다.

- `.sidebar` (271~285행 근처): width 64px, 호버 시 244px 등
- `.sidebar nav`, `.sidebar:hover span` 등
- `.logo`, `.menu`, `.menu-item`, `.menu-avatar` 등

나중에 “호버 시 넓어지는” 동작을 **공통 sidebar.css**에 다시 넣을 계획이면, 그때 `.sidebar` / `.sidebar__nav` 등으로 옮기면 됩니다.

### 2-3. 모바일에서 공통 하단 nav와 맞추기

지금 index는 `.mobile-bottombar`로 하단바를 쓰고, 공통은 `#bottom-nav` + `.bottom-nav`입니다.  
`mobile.css`가 이미 `#bottom-nav`를 모바일에서만 보이게 하고, `body { padding-bottom: 60px }` 등으로 처리하므로, **index에서 `.mobile-bottombar`를 제거하고 `#bottom-nav`만 쓰면** feed.css의 `.mobile-bottombar` 관련 규칙은 더 이상 필요 없습니다.  
필요하면 `@media (max-width: 768px)` 안의 `.mobile-bottombar { display: flex }` 같은 부분만 제거해도 됩니다.

---

## 3. script.js

**수정할 필요 없음.**  
이미 다음이 들어 있어서, `#sidebar` / `#bottom-nav`만 있으면 동작합니다.

```js
const sidebarEl = document.getElementById('sidebar');
if (sidebarEl) {
  fetch('sidebar.html')
    .then(res => res.text())
    .then(html => { sidebarEl.innerHTML = html; });
}
const bottomNavEl = document.getElementById('bottom-nav');
if (bottomNavEl) {
  fetch('bottom-nav.html')
    .then(res => res.text())
    .then(html => { bottomNavEl.innerHTML = html; });
}
```

---

## 4. sidebar.html / bottom-nav.html / mobile.css

- **sidebar.html**: 메뉴 순서·아이콘·링크만 맞추면 됨. index에 있던 “홈, 릴스, 메시지, 검색, 탐색 탭, 알림, 만들기, 프로필” 순서나 아이콘을 따오고 싶으면 이 파일만 수정.
- **bottom-nav.html**: explore/reels와 동일한 구조 그대로 써도 되고, index에서 쓰던 아이콘/순서로 바꾸고 싶으면 여기만 수정.
- **mobile.css**: `#bottom-nav` 스타일이 이미 있으므로 그대로 두면 됨. (필요하면 z-index, 높이 등만 조정)

---

## 5. 정리 체크리스트

1. **index.html**
   - head에 `sidebar.css`, `mobile.css` 링크 추가
   - 왼쪽 사이드바 전체 → `<div id="sidebar"></div>` 로 교체
   - 모바일 하단바 전체 → `<div id="bottom-nav"></div>` 로 교체
2. **feed.css**
   - `.main-container`에 `padding-left: 244px` (데스크톱) 추가
   - 기존 `.sidebar` / `.logo` / `.menu` / `.menu-item` 등 인라인 사이드바 전용 스타일 제거 또는 주석
   - `.mobile-bottombar` 관련 규칙 제거 또는 유지(사용 안 하면 제거)
3. **script.js**  
   - 변경 없음
4. **교체 후**
   - index에서 쓰고 싶었던 기능(호버 확장, 메뉴 순서 등)을 **sidebar.html / sidebar.css** 한 군데만 수정해서 공통으로 반영

이 순서로 하면 index도 공통 사이드바/하단 nav로 통일되고, 이후에는 공통만 고치면 됩니다.
